Race Condition
=====================================================

Current Problem:

If a transaction begins processing before the analyst blocks the user, it can still complete.

Fix:

Introduce a final authorization checkpoint.

Transaction lifecycle should become:

Receive Request

↓

Validate

↓

Fraud Score

↓

Review

↓

FINAL BLOCKLIST VALIDATION

↓

Execute

↓

Completed

The system MUST re-check the blocklist immediately before committing any transaction.

If blocked:

Abort transaction

Rollback if necessary

Mark transaction REJECTED

Emit websocket event

Return HTTP 403

No blocked transaction should ever succeed.
